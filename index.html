Differences between 'switch' and 'if/else'

First, let's define 'switch' and 'if/else':

if/else: When using 'if/else', you're telling the program that if the statement was true then continue down the program, but if the statement was not true then execute the 'else' statement. Once a statement is true, the program will exit the comparison. if/else will check for values in a range.

switch: A switch statement will run through the list of cases looking for a match, and will compare to every case statement until it hits a break. It will also only check data that is prespecified and equal to.

When you would use one or the other: Using 'if/else' is the better option when checking for values in a range and is ideal when trying to select between two options. Using 'switch' is the better option when there's multiple options and you need the statement to spit out a value that matches.

Using '=' vs '==' vs '.equals'

What does each of these do?

=: '=' is used to assign values to variables.

==: '==' is used to compare variables to a constant to check for equality. Typically used for conditional statements. Can only be used with integers and doubles.

.equals: This will compare a string to another string and will only allow true or false. '.equals' is the best way to compare Strings for equality.

What is Math?

Math allows a program or application programmed in Java to execute various mathematic operations.

What is the return type of the ‘.random()’ method?

The typical return type for the '.random()' method is a double between 0.0 and 1.0.

Why is the 'M' in 'Math' capitalized?

The 'M' is capitalized since 'Math.random()' is a class. It is also proper etiquette for other programmers in order to more easily read your code.

'for', 'while', and 'do while' loop differences

'while' loop: Will repeat as long as a given condition/expression is true, it will also only execute if the condition is true. 'while' loops are best used when you are building an application where you are looking for a specific condition to be met

'do while' loop: Although similar to a 'while' loop, a 'do while' will always execute the body of the loop at least once, the condition/expression is then checked at the end of the loop. 'do while' loops are best used when you need to execute a loop to check what's in the condition. The benefit of a 'do while' loop is to ensure a block of code is executed at least once.

'for' loop: 'for' loops are designed to be executed a specified number of times as opposed to 'while' and 'do while' loops which will run as long as something is true. 'for' loops are best used when you know ahead of time how many iterations the loop needs to perform. A 'for' loop could be used when trying to assign numbers to a given number of people.

import java.util.Scanner;

When you include import java.util.Scanner; at the top of your code, you're allowing it access to get information from the keyboard, files on a computer, or the internet.

This functionality comes from a hierarchy. It is importing various "folders" - starting with the java folder, then the util or "utility class" folder, and ending with the Scanner class that enables the code to take inputs from a user.

'void' in a method declaration

What is it?: 'void' in a method declaration means that the method does not return a value. A method with a return type will return a value, whether it be an int, double, string, etc.
